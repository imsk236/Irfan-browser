# Omani Manuscript Archive — Domain Glossary

## Canonical terms

| Arabic | English gloss | Notes |
|---|---|---|
| عنوان (pl. عناوين) | Work / text | A text contained within a volume. The DB table is `works`, but all UI labels use **عنوان**. Do not use **أثر** in the UI. Tracks: العنوان (title), رقم الجزء (part number, optional), الأوراق (folio range), مؤلف, ناسخ (one or more), المساهم (zero or more), مصدر الصلة, تاريخ النسخ, ملاحظات. `work_type` removed. |
| مؤلف at work level | Author (work-level) | Exactly one person (or **مجهول**) per عنوان — required, cannot be left blank. `link_person_to_work` rejects linking a second, different person as مؤلف on the same work; this restriction is intentional and is not relaxed by المساهم (see ADR 0006). Carries its own مصدر الصلة. |
| ناسخ at work level | Scribe (work-level) | One or more persons per عنوان — required: the list must be non-empty, or **مجهول** must be checked. **مجهول** is only offered while the list is empty, and disappears once a named ناسخ is added (mutually exclusive with the list). Each carries its own مصدر الصلة. |
| المساهم at work level | Contributor (work-level) | Optional, zero or more persons per عنوان, each assigned a role from a dedicated `contributor_role` vocabulary: الراوي، المترجم، الجامع، المرتب، المعلق، المستدرك، المصحح، مؤلف مشارك — or **غير ذلك** (free text, not persisted to vocab, per ADR 0002). Distinct from the `role` vocab used by ناسخ/مالك/مستعير/etc. **مؤلف مشارك** is the sanctioned way to credit joint authorship; it does not reopen the single-مؤلف restriction (see ADR 0006). Each carries its own مصدر الصلة. Always referred to as **المساهم**, never "contributor," in code and UI. |
| منسوخ له at work level | Copy recipient (work-level) | Optional work-level role. The person for whom this copy was made, when stated in the colophon. |
| مصدر الصلة | Evidence source (how we know) for a work-level relationship | Per-relationship radio choice: المخطوط، المفهرس، or مرجع آخر (free text). Hardcoded in `WorkForm.tsx`/`PersonRoleSection.tsx`, not vocab-backed. The `knowledge_source` vocab category still exists in the DB (seeded: المجلد، مصدر خارجي، المفهرس) but is not currently wired to any UI — do not confuse the two. Every مؤلف/ناسخ/المساهم row carries a مصدر الصلة. |
| مصدر العنوان | Title source | How the cataloguer determined the title of the عنوان. Same three radio options as مصدر المعرفة (المخطوط، المفهرس، مرجع آخر). Stored as `title_source` on the `works` table. |
| رقم الجزء | Part number | Optional integer: which part/volume of a multi-part work this عنوان represents (long books are often copied across several أجزاء). Stored as `part_number` on the `works` table. Shown directly under العنوان in WorkForm. |
| مكان النسخ | Copy place (work-level) | The Omani wilaya where this copy was made. Single value: either a specific wilaya, **مجهول**, **خارج عُمان**, or unset (not recorded). Stored as `copy_place` on the `works` table. |
| تاريخ النسخ | Copying date (work-level) | Structured Hijri date on عنوان. Five components, all optional (blank shown as **مجهول** placeholder, stored as NULL in DB): السنة (INTEGER), الشهر (TEXT — 12 fixed Hijri months), التاريخ (INTEGER 1–30, day of month), اليوم (TEXT — 7 fixed weekdays), الوقت (TEXT — free text). Plus: التاريخ كما هو مكتوب (TEXT witness, verbatim) — stored on the model but not currently exposed in WorkForm's UI. |
| مجلد (pl. مجلدات) | Volume | A physical manuscript codex. |
| قيد (pl. قيود) | Annotation | A physical inscription on the manuscript (ownership mark, reading certificate, etc.). Stored in `annotations` table. Work link labeled **العنوان**, dropdown with **ليس مرتبط بعنوان معين** as default. Person relationships at the volume level are always entered through a قيد — there is no standalone "الأشخاص المرتبطون" section on the volume. **نوع القيد** is a dedicated `annotation_type` vocab (إعارة، إهداء، تملُّك، سماع، شراء، مطالعة، معارضة، مقابلة، نظر، هِبَة، وقف). The قيد form allows adding multiple persons, each with a role selected from a dedicated `role` vocab (ناسخ، كاتب، مُعير، مستعير، مُهْدِي، مُهْدَى إليه، مالك، شاري، بائع، مُطالِع، مُعارض، معروض عليه، مُقابِل، ناظر، واهب، موهوب، واقف، موقوف، ناظر الوقف، دلال، شاهد). No conditional filtering by قيد type — same role dropdown always available. **مؤلف ومنسوخ له are excluded from this dropdown** — those two are work-level-only roles assigned exclusively via WorkForm's dedicated slots, never through a قيد. **ناسخ is allowed through a قيد** (e.g. a later inscription separately naming a copyist), distinct from the work-level ناسخ assignment. All person relationships (whether from عنوان or قيد) must appear together in the trace/search screen. Both **نوع القيد** and **الدور** offer a **غير ذلك** option revealing a free-text field — a custom value entered this way is stored verbatim on that record only and is *not* added to the vocab list (won't appear as a selectable option on future قيود). A person may be linked to the same قيد/volume with two or more distinct roles simultaneously. |
| شخص (pl. أشخاص) | Person | A normalized identity record. Tracks: الاسم المعتمد (required), name components (إسم, كنية, لقب, نسبة ×2, معروف بـ), سلسلة النسب (free-text nasab chain), birth/death dates (as written), birth/death places (free text), المنطقة (zero or more Omani wilayas via `person_wilayas` junction table, or the sentinel **مجهول**, or the option **خارج عُمان**), ملاحظات. Fields intentionally absent from the form: حالة التعريف, الانتماء العلمي, المهنة أو الصفة, صيغ الأسماء. |
| خزانة (pl. خزائن) | Repository | The institution or library holding the volume. Use **خزانة** in all UI labels. Do not use **مستودع**. Tracks: اسم الخزانة, الولاية (Omani wilaya dropdown), مفتاح الخزانة (4-digit PPPP code), ملاحظات. `kind` field dropped. |
| رقم الخزنة | Document number | Our internal sequential number per خزانة (`document_number` in DB). The DDDD component of the serial. Auto-assigned at creation and pre-filled as an editable field in the creation form only — the researcher may change it before first save. Not editable after the volume is saved. Do not use "رقم الوثيقة". |
| رقم المجلد في الخزانة | Repository volume number | The integer reference the external library physically put on the manuscript (`repository_volume_number` in DB). Always a plain integer. |
| الرقم التسلسلي | Serial | Read-only PPPP-DDDD badge assembled from مفتاح الخزانة + رقم الخزنة. Never hand-typed. |
| الرقم التسلسلي / السيريال | Serial | Human-readable `PPPP-DDDD` badge. Auto-generated; never hand-typed. |
| المطلع | Incipit | Verbatim opening words of the work as written in the manuscript. A witness field (`as_written`). Used to identify the work when the title is absent or ambiguous. |
| الخاتمة | Explicit | Verbatim closing words of the work as written in the manuscript. A witness field (`as_written`). |
| التصنيف الموضوعي | Subject classification | A two-level topic assigned to a عنوان: التقسيم العام (one of 12 main categories) and التقسيم الفرعي (subcategory within that category). Single classification per عنوان — no multi-tagging. |
| شاهد / كما هو مكتوب | Witness (`as_written`) | Verbatim transcription from the page. Immutable evidence. |
| تفسير / تأويل | Interpretation | A researcher conclusion drawn from a witness. No confidence level — confidence tracking has been removed from the data model. |
| ورقة (pl. أوراق) | Folio | A leaf of a manuscript. Each ورقة has two faces: **يمين** (right face, the one you encounter first in RTL reading) and **يسار** (left face). Physical order: 1ي → 1س → 2ي → 2س … Encoded in the system as `{n}ي` or `{n}س` (e.g. `15ي`, `24س`). |
| يمين / يسار | Recto / verso (folio face) | The two faces of a ورقة. يمين always precedes يسار on the same folio number. In all folio input widgets, يمين is the default selection and the dropdown is dimmed until a folio number is entered. |
| موضع اللوحة | Folio location (annotation) | A single folio face reference on a قيد, encoded as `{n}ي` or `{n}س`. Not a range. |
| تاريخ القيد | Annotation date | Structured Hijri date on قيد, same five components as تاريخ النسخ (السنة, الشهر, التاريخ, اليوم, الوقت) and same optional/مجهول-placeholder behavior — but **no مكان field** (قيود don't record where they were inscribed; موضع اللوحة already answers "where in the book," and there's no established practice of recording geographic place of inscription) and **no separate witness field** (النص كما هو مكتوب already captures the date verbatim as part of the full inscription text). Reinstates dates on قيود, previously removed entirely in migration `007_annotation_drop_date_fields` — see ADR 0003. |
| البحث والتتبع (unified search) | Unified search | **البحث والتتبع** is a single search — not a choice of modes. One filter panel holds every criterion, all optional, all combinable (AND across categories): شخص (a specific person, via the person-picker autocomplete), منطقة العالم (a person's own region, via `person_wilayas` — distinct from مكان النسخ), العنوان (matches `Work.title` only — not المطلع/الخاتمة), الرقم (one merged field, matches either الرقم التسلسلي substring or رقم المجلد في الخزانة), خزانة, مكان النسخ (a work's copying place — a wilaya, distinct from منطقة العالم), and سنة النسخ range (matches if a linked عنوان's `copy_year` **or** a linked قيد's `annotation_year` falls in the range). At least one criterion is required. Results are **appearance-shaped**: one row per matching `PersonRelationship` (role, evidence, serial, عنوان), same grain as the old بحث عن عالم output — *not* one row per volume. A مجلد/عنوان that matches on title/number/خزانة/date/مكان النسخ but has no relationship satisfying the (possibly absent) person criteria still surfaces, as a placeholder row (no role/evidence) — this is the only case a row isn't a real relationship. When a year range is active, works/annotations with no recorded year are excluded. Ranked/aggregate browsing (scholars ranked by appearance count, خزائن ranked by volume count within a region) is dropped — البحث والتتبع finds specific things matching criteria, it does not rank or browse. See ADR 0005 (supersedes ADR 0004, which described the now-retired standalone بحث عن مجلد mode). |
