# Product

## Register

product

## Users

A single researcher (Irfan) using a personal desktop app to catalog ~400 Omani manuscript volumes and trace scholars across them. Works in Arabic, thinks in Arabic manuscript tradition, needs precision tools that disappear into the work.

## Product Purpose

Omani manuscript archive manager — catalog volumes, record works within them, trace scholars (persons) across volumes and wilayas. Success means the researcher can find, cross-reference, and export any piece of data without fighting the interface.

## Brand Personality

محترف · نظيف · فعّال — Professional, clean, efficient. The UI serves the data; it does not perform. The Arabic calligraphic identity (عرفان mark, dark green palette) signals domain authenticity without ornamentation for its own sake.

## Anti-references

- Generic SaaS dashboards (Notion, Airtable, gradient tiles, metric hero cards) — no domain identity
- Dated government archival software — heavy tables, no hierarchy, Windows-XP-era form styling

## Design Principles

1. **Serve the work** — every UI element earns its place by helping the researcher. Decoration that doesn't aid comprehension is removed.
2. **Domain-authentic** — the visual language reflects manuscript scholarship: measured, precise, unhurried.
3. **Consistency above novelty** — new screens must feel like they belong to the existing system. Match spacing tokens, component patterns, and typographic hierarchy from the established screens.
4. **RTL-first** — all spacing, layout, and directional logic is written for Arabic right-to-left reading.
5. **Data over chrome** — the manuscripts are the product. The interface is the container.

## Accessibility & Inclusion

RTL Arabic throughout. Single-user desktop Electron app; no public WCAG requirement, but text contrast and keyboard navigation are expected. Reduced motion is not currently addressed.
